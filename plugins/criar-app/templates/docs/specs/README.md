# Specs

Uma spec por funcionalidade (SPEC-00N), gerada a partir das **regras de negócio**. Crie/atualize com o comando **/spec <funcionalidade>**.

- Toda spec **lista as regras (RN-00x)** que a sustentam — assim código, spec e regra ficam rastreáveis.
- `status`: rascunho → aprovada → implementada → obsoleta.
- A spec é o contrato que o código deve cumprir; o subagente **revisor** confere o código contra ela.
